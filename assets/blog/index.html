Introducing Numbat

Numbat is a programming language with a type system that treats physical dimensions as types. The type checker infers types for every expression in the program and ensures that everything is correct in terms of physical dimensions — and therefore also correct in terms of units. For example, the expression 3 months has a type of Time. The expression 2 years also has a type of Time. The compound expression 3 months + 2 years is therefore well-typed.
On the other hand, 3 months + 2 lightyears is ill-typed, because the right-hand side is of type Length. Try to change years to lightyears in the example below to see the resulting compiler error:

3 months + 2 years


    

    Basing the type system on physical dimensions instead of physical units has a few advantages. For one, we do not have to change type annotations when switching units.

The following program calculates and prints the flow rate in a pipe
using the Hagen-Poiseuille equation. It assumes the dynamic
viscosity of water and allows for inputs of pipe radius,
pipe length, and pressure difference.


    
let μ_water: DynamicViscosity = 1 mPa·s

fn flow_rate(radius: Length, length: Length, Δp: Pressure) -> FlowRate =
    π × radius^4 × Δp / (8 μ_water × length)

let pipe_radius = 1 cm
let pipe_length = 10 m
let Δp = 0.1 bar

let q = flow_rate(pipe_radius, pipe_length, Δp)
